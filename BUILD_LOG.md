# Freedao 构建日志

> Build in Public 素材库。记录产品从 0 到 1 的关键决策、技术选型、进展里程碑。

## 2026-04-19

### 产品定位

**一句话**：独立创业者（OPC）的现金流生存仪表盘。不是记账 App，是"距离自由还差多少"的倒计时。

**核心洞察**：大多数 AI 时代的独立创业者还没有收入，靠积蓄撑着，但不知道自己还能撑多久。Freedao 把这个焦虑量化成一个倒计时数字，让"不确定"变成"可计划"。

**目标用户**：全职独立创业者 / AI OPC 探索者，还没有收入或收入不稳定，靠积蓄+家人资助撑着。

### 核心差异化

| 记账 App | Freedao |
|---|---|
| 告诉你"钱花哪了" | 告诉你"还差多少钱才自由" |
| 记录过去 | 规划未来 |
| 面向所有人 | 面向独立创业者 |
| 手动分类 | AI 自动分类，用户可修正 |
| 功能：分类记账 | 功能：跑道计算 + 时间追踪 + AI 建议 |

### 设计过程

1. **Claude Design 原型**（约 10 分钟）
   - 用 Claude Design 生成高仿真 React 原型
   - 设计系统：深色主题 + Inter/Instrument Serif/JetBrains Mono 三字体 + oklch 色盘
   - 品牌色选择 amber（琥珀），传达"自由的暖意"
   - 效果炸裂——看到原型就想拥有

2. **MVP 代码初版**（约 15 分钟）
   - Next.js + TypeScript + Tailwind CSS
   - 功能完整但视觉很粗糙
   - "看完就没有购买欲望"

3. **Design Handoff**（约 20 分钟）
   - 从 Claude Design 导出 handoff bundle
   - 直接在 MVP 代码上匹配原型的视觉设计
   - 96px 衬线大数字 + 渐变进度条 + "你在这里"发光标记
   - 视觉效果从"丑爆了"升级到"想拥有"

### 技术选型

| 层面 | 选择 | 原因 |
|---|---|---|
| 框架 | Next.js (App Router) | SSR + API Routes + Vercel 一键部署 |
| 语言 | TypeScript | 类型安全 |
| 样式 | Tailwind CSS + 内联样式 | 匹配设计稿的精确值 |
| 数据 | localStorage | v1 无后端，零运维 |
| AI | 多模型（DeepSeek/Gemini/Qwen/火山引擎/自定义） | 用户 BYOK，成本自控 |
| 部署 | Vercel | 免费额度足够 MVP |

### 关键决策记录

1. **为什么不做后端？** v1 数据存本地，避免合规负担（涉及用户财务数据）。等验证通过再考虑多设备同步。

2. **为什么支持多模型而不只用一个？** 面向 OPC 用户，他们可能已有不同平台的 API Key。降低使用门槛，BYOK 模式零成本。

3. **支出分类为什么用 AI？** 手动选"投资/消费/必选/可选"太繁琐。AI 自动判断 + 用户一键确认，录入效率提升 3x。

4. **为什么先做"时间追踪"而不是"CSV导入"？** 对 OPC 来说，时间 = 唯一生产资料。看清"直接产生收入的时间占比"比导入历史账单更紧迫。

### 功能模块

- **Dashboard**：全景仪表盘——倒计时 + 收支结构 + 时间分配
- **Log**：记录流水——快速录入 + AI 分类 + 时间线展示
- **Plan**：规划模拟——目标设定 + what-if 场景（v2）
- **Settings**：配置——模型选择 + API Key + 数据管理

### 验证方式

- Build in Public 视频介绍产品思路和功能
- 飞书表单收集用户使用意向
- 24h 内 ≥10 人留联系方式 = Strong signal → 继续投入
- 沉默 → 转为自用轻量版，不继续公开推广
